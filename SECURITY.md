# Security Policy

## Security

We take security seriously. Our products are built on open-source cryptographic libraries with `#![forbid(unsafe_code)]`, post-quantum key encapsulation (ML-KEM-768/1024, FIPS 203), and zero-knowledge architecture.

## Reporting a Vulnerability

If you find a vulnerability in any Moss Piglet project:

- Report it directly to the repository where you found it (via GitHub's private vulnerability reporting if available)
- If you're unable to report there or receive no response, contact us at security @ mosspiglet . dev

We are a small team but we take every report seriously and will respond as quickly as we can.

Thank you for helping keep our users safe.
